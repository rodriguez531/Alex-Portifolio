9.2.5Generate the HTML File

In the previous step, we created the HTML string from a function. Now we need to create an HTML file using that HTML template literal.

Do some research on how to create a file. Keep in mind that nearly all programming languages have an interface to the computer's file system. The file system is the part of a computer's operating system that handles reading and writing files to persistent storage, such as a hard drive. File system interfaces are prevalent because organizing and accessing files has always been an essential task of computers.

To learn more about how Node.js interfaces with the file system, read Node.js's official documentation (Links to an external site.).

You can also learn more about the Node API in the following video:

In this documentation we can find the newest features available to us as well as bug fixes, instructions, examples and usage cases.

important

To ensure the documentation syncs to the version of Node.js you're running on your computer, type node --version in your terminal.

In the table of contents of the documentation, look for the section named File System. The file system is considered a module in Node.js's core library. A module can be a function, a class, an object, or simple variables. Whatever its form, a module is a reusable piece of code that can be imported to anywhere it's needed. The core library in Node.js contains many helpful modules, like a utility module that can assist with directory path names, or even an interface with the Console, which is a crossover from the browser and the Web API, which we use frequently.

Although there are many useful modules in the core library, most lie beyond the scope of this lesson. Let's focus on the file system module. Select the File System link in the table of contents. We'll be directed to another list that details what the File System interface in Node.js is capable of.

A keyword search for the word "writefile" should lead you to a function called fs.writeFile().

According to the documentation and the examples, this function can create multiple file types, including TXT, PDF, HTML, JSON, and more. The fs.writeFile() function definition has three arguments. The first argument is the name of the file that's being created. The next argument is the data that will write onto the file, in this case the HTML template literal. The last parameter is a callback function that will be used for error handling.

important

In the fs module, all functions have synchronous and asynchronous versions, including fs.writeFile() and fs.writeFileSync(). To learn more, see Node.js's official documentation (Links to an external site.).

Go to the app.js file, delete both the console.log(name, github); and the console.log(generatePage(name, github)); statements, and then type the following at the bottom of the file:

fs.writeFile('index.html', generatePage(name, github), err => {
  if (err) throw err;

  console.log('Portfolio complete! Check out index.html to see the output!');
});

Review the three arguments in the fs.writeFile() function. The first argument is the file name that will be created, or the output file. The second argument is the data that's being written: the HTML string template. The third argument is the callback function that will handle any errors as well as the success message.

pause

What are the advantages of having an asynchronous function handle a file-writing process?

Writing data to a file can strain a computer's resources. Handling this process asynchronously allows the computer to run the process in the background, freeing up resources to continuously execute the current program without halting the runtime.

First, look at the arrow function. Notice a few differences from the arrow function in the last step. In this case, no parentheses enclose the argument. We described the rule about this syntax once before, but it bears repeating:

When an arrow function has one argument, parentheses are optional. However, when there are no arguments—or more than one—parentheses are necessary.

In the callback function block, a conditional statement checks for the err being returned by the callback function. If err exists, an error message is displayed, using the following statement:

if (err) throw err;

Rather than silently displaying the error with console.log(err);, the preceding statement creates an exception and stops the execution of the code.

See the following image for an example of an error message:

Notice the message directly beneath the line of code that generated the error. The message reads:

TypeError [ERROR_INVALID_CALLBACK]: Callback must be a function. Received undefined

This message concatenates two text properties of the Error object that is returned, name and message. In this case, name is "TypeError", and message is "Callback must be a function. Received undefined." It's important to know this, for you may wish to retrieve properties of the Error object while you're developing your applications, and these two are central to identifying problems.

Back in our code, after the if (err) throw err; statement is a console.log() success statement that directs users to inspect the newly created file.

It's time to check our work. Save the file changes and execute the Node.js app. This time, we'll encounter the error message shown in the following image:

This error message indicates that the fs module is not defined or could not be found.

pause

What are the name and message properties of the Error object, referenced in the preceding image?

The value of the name property is ReferenceError. The value of the message property is "fs is not defined."

Look at the documentation to see if any instructions were missed. The instructions shown in the following image appear under the table of contents:

The instructions state that in order to use the fs module, we need the following statement at the top of the JavaScript file. Let's add this statement to the top of the app.js file and then discuss its significance:

const fs = require('fs');

The require statement is a built-in function that's globally available in Node.js. It allows the app.js file to access the fs module's functions through the fs assignment.

deep dive
deep dive

Node.js follows the CommonJS module system and uses require to include modules, which can be any reusable piece of code from the core library or even a separate JavaScript file. To learn more, see the Wikipedia article on CommonJS (Links to an external site.). For further information, read the Node.js documention on modules (Links to an external site.).

Let's watch a quick video that will step us through how this module system can help import and export data into our applications:

It is a common misconception that because modules like fs are a part of Node.js's core library, they're globally available. They're not. Using Node.js's modular system, we can selectively choose which modules we need (using require) and avoid slowing down our application with modules we don't.

Let's try to run the app again to see if this fixed the ReferenceError we received earlier. Don't forget to pass the name and github values in as arguments.

We should receive a message like the following image from the terminal after saving the changes and running the application:

This message indicates that the index.html file was created, and we can see that a file called index.html now exists at the root of our application. We just created an entire HTML file with only JavaScript!
